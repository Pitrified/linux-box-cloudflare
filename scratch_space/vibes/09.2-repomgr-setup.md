### 2. `repos.toml`

Location: put it in `linux-box-cloudflare/configs/repomgr`, as it is a custom config for this workspace, not a general repomgr config. The repomgr package will be configured to look for it there.

All remotes use SSH (`git@github.com:...`) since we have ssh access to the repos.

```toml
# repos.toml - repomgr config for the linux-box workspace

[settings]
base_path        = "~/repos"
default_test_cmd = "uv run pytest"
state_file       = "./repos.state.json"

# ---------------------------------------------------------------------------
# Sources - produce versioned tags consumed by other repos
# ---------------------------------------------------------------------------

[[repo]]
name       = "llm-core"
remote     = "git@github.com:Pitrified/llm-core.git"
roles      = ["source"]
auto_merge = true

[[repo]]
name       = "fastapi-tools"
remote     = "git@github.com:Pitrified/fastapi-tools.git"
roles      = ["source"]
auto_merge = true

# Source + consumer: depends on llm-core + fastapi-tools; consumed by kit-hub
[[repo]]
name       = "media-downloader"
remote     = "git@github.com:Pitrified/media-downloader.git"
roles      = ["source", "consumer"]
auto_merge = true

# ---------------------------------------------------------------------------
# Consumers - depend on git-sourced packages from the sources above
# ---------------------------------------------------------------------------

[[repo]]
name   = "laife"
remote = "git@github.com:Pitrified/laife.git"
roles  = ["consumer"]

[[repo]]
name   = "kit-hub"
remote = "git@github.com:Pitrified/kit-hub.git"
roles  = ["consumer"]

# ---------------------------------------------------------------------------
# Standalone - tracked for git status/fetch only, not in dep chain
# ---------------------------------------------------------------------------

[[repo]]
name   = "linux-box-cloudflare"
remote = "git@github.com:Pitrified/linux-box-cloudflare.git"
roles  = ["consumer"]

[[repo]]
name   = "repomgr"
remote = "git@github.com:Pitrified/repomgr.git"
roles  = ["consumer"]

[[repo]]
name   = "tg-central-hub-bot"
remote = "git@github.com:Pitrified/tg-central-hub-bot.git"
roles  = ["consumer"]

[[repo]]
name   = "python-project-template"
remote = "git@github.com:Pitrified/python-project-template.git"
roles  = ["consumer"]

[[repo]]
name   = "recipamatic"
remote = "git@github.com:Pitrified/recipamatic.git"
roles  = ["consumer"]

[[repo]]
name   = "recipinator"
remote = "git@github.com:Pitrified/recipinator.git"
roles  = ["consumer"]

[[repo]]
name   = "cookbook"
remote = "git@github.com:Pitrified/cookbook.git"
roles  = ["consumer"]

[[repo]]
name   = "convo_craft"
remote = "git@github.com:Pitrified/convo-craft.git"
roles  = ["consumer"]
```

Notes:
- `auto_merge = true` only on sources (llm-core, fastapi-tools, media-downloader) because those are the ones that get auto-bumped by `update-deps`. Consumers stay `false` (default) - updates go to a branch for manual review.
- `convo_craft`: the remote is `convo-craft` (hyphen), not `convo_craft` (underscore) - careful there. UPDATE: will this break `repomgr` ?
- `recipamatic` uses poetry (not uv), so `uv run pytest` will fail. Either set `test_cmd = "cd py && poetry run pytest"` or leave test_cmd unset and skip testing for it. Since `update-deps` only targets consumers with git deps, and recipamatic has none, it is just tracked passively. UPDATE: ok to leave it without testing for now.

### 3. Docs section in `linux-box-cloudflare/docs/`

Create a new file: `docs/02_repomgr.md` (or next available number).

Sections to cover:

1. **What repomgr does** - short description, link to the repomgr repo.
2. **Installation** - repomgr is cloned at `~/repos/repomgr`; run `uv sync` there.
3. **Config location** - `repos.toml` lives at `~/repos/linux-box-cloudflare/configs/repomgr/repos.toml` (not inside the repomgr package, since it's a custom config for this workspace). Explain how to point repomgr to this config if needed.
4. **Common commands**:
   - `repomgr status` - health dashboard across all repos
   - `repomgr fetch` - fetch all remotes, report divergences
   - `repomgr clone-missing` - clone any repo not yet on disk
   - `repomgr dep-graph` - print the source/consumer dependency tree
   - `repomgr update-deps` - bump git deps and run tests; auto-merges sources
   - `repomgr stale-branches` - list and interactively delete old branches
5. **Dependency graph** - diagram of which repos consume which.
6. **Adding a new repo** - append a `[[repo]]` entry to `repos.toml`, then `repomgr clone-missing`.

### 4. Implementation steps

1. Create `~/repos/linux-box-cloudflare/configs/repomgr/repos.toml` from the template above.
2. Run `repomgr status` to verify all repos are found.
3. Convert any remaining HTTPS remotes to SSH (`git remote set-url origin git@github.com:...`) for the repos that still use HTTPS.
4. Create `linux-box-cloudflare/docs/02_repomgr.md` (or next doc number).
5. Update `linux-box-cloudflare/docs/index.md` or nav if one exists to include the new doc.
