# Audio Extraction for Transcription a-transcribe Design

## Context

`llm-core` transcription providers accept a `Path` to an audio file.
Downloaded media from yt-dlp and Instagram is typically a video (`.mp4`, `.webm`).
Audio extraction is the downloader's responsibility, sitting inside
`TranscriptionHook.process()` before the call to `transcriber.transcribe()`.

The extraction function is already proven in both recipamatic and recipinator notebooks,
where it appears as:

```python
def extract_audio(video_fp: Path, audio_fp: Path, audio_format: str = "mp3") -> None:
    video_format = video_fp.suffix[1:]
    video = AudioSegment.from_file(video_fp, format=video_format)
    audio = video.set_channels(1).set_frame_rate(16000).set_sample_width(2)
    audio.export(audio_fp, format=audio_format)
```

The three parameters a-transcribe mono (`set_channels(1)`), 16 kHz (`set_frame_rate(16000)`),
16-bit PCM (`set_sample_width(2)`) a-transcribe match exactly what Whisper and faster-whisper
expect internally. They would normalise the audio anyway; doing it explicitly here
produces a smaller, faster-loading file.

---

## Format choice: `wav` for local providers

Both notebooks use `mp3`. For a local provider this is suboptimal:
- `mp3` is lossy and requires a re-encode on export
- `wav` is lossless PCM, no re-encoding step, faster write and load
- `wav` size is not a concern since files are local and temporary

For `OpenAIAPITranscriber`, upload size does matter. `mp3` is the right default there.
The extraction format should therefore be configurable per provider context, defaulting
to `wav`.

---

## Where extraction lives

`TranscriptionHook` is the right owner. It sits between the download result and the
transcriber call, and it is the only place that knows both "what file was downloaded"
and "what the transcriber needs".

A dedicated `AudioExtractor` class is extracted from the hook logic for two reasons:
- it is independently testable without a full `DownloadedMedia`
- other future consumers (recipamatic migration) can import it directly

## `AudioExtractor`

```python
# post_processing/audio_extractor.py

from __future__ import annotations

import asyncio
from pathlib import Path

from loguru import logger as lg


# File extensions that can go directly to a transcriber without extraction.
# pydub can read all of these, but passing them through unchanged saves
# a re-encode cycle.
_AUDIO_EXTENSIONS: frozenset[str] = frozenset({
    ".mp3", ".wav", ".m4a", ".aac", ".ogg", ".flac",
    ".mpga", ".mpeg",
})

# Video extensions that require audio extraction before transcription.
_VIDEO_EXTENSIONS: frozenset[str] = frozenset({
    ".mp4", ".webm", ".mkv", ".mov", ".avi", ".ts",
})


def needs_extraction(fp: Path) -> bool:
    """Return True if the file is a video that requires audio extraction."""
    return fp.suffix.lower() in _VIDEO_EXTENSIONS


def extract_audio(
    video_fp: Path,
    audio_fp: Path,
    *,
    channels: int = 1,
    frame_rate: int = 16_000,
    sample_width: int = 2,
    audio_format: str = "wav",
) -> Path:
    """Extract and normalise audio from a video file using pydub.

    The three normalisation parameters match Whisper's expected input:
    - channels=1       mono
    - frame_rate=16000 16 kHz sample rate
    - sample_width=2   16-bit PCM (2 bytes per sample)

    Doing this explicitly rather than letting Whisper normalise internally
    produces a smaller, faster-loading audio file.

    Args:
        video_fp:     Path to the source video file.
        audio_fp:     Destination path for the extracted audio.
                      Parent directory must already exist.
        channels:     Number of output channels. Defaults to 1 (mono).
        frame_rate:   Output sample rate in Hz. Defaults to 16000.
        sample_width: Bytes per sample. 2 = 16-bit PCM.
        audio_format: Output format passed to pydub export.
                      "wav" for local providers, "mp3" for OpenAI API.

    Returns:
        audio_fp (for chaining convenience).

    Raises:
        ImportError:  If pydub is not installed.
        FileNotFoundError: If video_fp does not exist.
        CouldntDecodeError: If pydub/ffmpeg cannot decode the source file.
    """
    from pydub import AudioSegment  # deferred: pydub is an optional dep  # noqa: PLC0415

    if not video_fp.exists():
        raise FileNotFoundError(f"Video file not found: {video_fp}")

    video_format = video_fp.suffix.lstrip(".").lower()
    lg.debug(f"Extracting audio: {video_fp} -> {audio_fp} ({audio_format})")

    video = AudioSegment.from_file(video_fp, format=video_format)
    audio = (
        video
        .set_channels(channels)
        .set_frame_rate(frame_rate)
        .set_sample_width(sample_width)
    )
    audio.export(audio_fp, format=audio_format)

    lg.debug(f"Audio extracted: {audio_fp} ({audio_fp.stat().st_size / 1024:.1f} KB)")
    return audio_fp


async def extract_audio_async(
    video_fp: Path,
    audio_fp: Path,
    **kwargs: object,
) -> Path:
    """Async wrapper: runs extract_audio in a thread pool.

    pydub wraps ffmpeg which is CPU/IO-bound. Running in asyncio.to_thread
    keeps the event loop free during extraction.

    Args:
        video_fp: Path to the source video file.
        audio_fp: Destination path for the extracted audio.
        **kwargs: Forwarded to extract_audio.

    Returns:
        audio_fp.
    """
    return await asyncio.to_thread(extract_audio, video_fp, audio_fp, **kwargs)
```

## Audio file sidecar strategy

Rather than a `tempfile.TemporaryDirectory`, extraction writes a persistent sidecar
alongside the video inside `MediaStorage`. For a post at
`{media_base_dir}/instagram/CsEj0n9Kefd/`, the layout becomes:

```
CsEj0n9Kefd/
├── video.mp4          ← downloaded by InstaDownloader
└── audio.wav          ← extracted by AudioExtractor, persisted
```

**Why persist rather than use a temp file:**

- Repeated transcription calls (e.g. re-transcribing with a different model) skip
  re-extraction a-transcribe the audio sidecar is reused if it already exists
- Consistent with how the notebooks worked: `p_audio.mp3` sat next to `p_video_url.mp4`
- `MediaStorage` already owns the folder; the sidecar is just another role

The `MediaStorage.media_path()` method already supports this via the `role` parameter:

```python
audio_fp = storage.media_path(source, source_id, role="audio", ext="wav")
```

The `MediaFile` DB row for the sidecar uses `role="audio"`, `media_on_disk=True`.
If the video is ever manually deleted, the audio sidecar can remain a-transcribe the
`media_on_disk` flag per-file handles this cleanly.

## `TranscriptionHook` (updated)

```python
# post_processing/transcription.py

from __future__ import annotations

import asyncio
from pathlib import Path
from typing import TYPE_CHECKING

from loguru import logger as lg

from social_media_downloader.post_processing.audio_extractor import (
    extract_audio_async,
    needs_extraction,
)

if TYPE_CHECKING:
    from llm_core.transcription.base import BaseTranscriber
    from social_media_downloader.core.models import DownloadedMedia
    from social_media_downloader.storage.media_storage import MediaStorage


class TranscriptionHook:
    """Post-processor: extracts audio if needed, then transcribes.

    Audio extraction and transcription are both CPU/IO-bound and run via
    asyncio.to_thread internally (inside extract_audio_async and
    atranscribe respectively). The hook's own public API is async-first.
    """

    def __init__(
        self,
        transcriber: BaseTranscriber,
        storage: MediaStorage,
        audio_format: str = "wav",
    ) -> None:
        """
        Args:
            transcriber:  A BaseTranscriber instance (from llm-core), created
                          once at service startup via create_transcriber().
            storage:      MediaStorage instance for resolving the audio sidecar path.
            audio_format: Extraction format. "wav" for local providers (default),
                          "mp3" for OpenAIAPITranscriber to reduce upload size.
        """
        self._transcriber = transcriber
        self._storage = storage
        self._audio_format = audio_format

    async def aprocess(self, media: DownloadedMedia) -> DownloadedMedia:
        """Extract audio if needed, transcribe, populate media.transcription.

        Skips gracefully if no video or audio file is present.
        Reuses an existing audio sidecar if already extracted.

        Args:
            media: DownloadedMedia to enrich in-place.

        Returns:
            The same DownloadedMedia instance with transcription populated.
        """
        audio_fp = await self._resolve_audio(media)
        if audio_fp is None:
            lg.debug(f"No audio available for {media.source_id}, skipping transcription")
            return media

        lg.info(f"Transcribing {audio_fp} via {self._transcriber.provider_name}")
        media.transcription = await self._transcriber.atranscribe(audio_fp)
        lg.info(
            f"Transcription done: {len(media.transcription.text)} chars, "
            f"language={media.transcription.language}"
        )
        return media

    def process(self, media: DownloadedMedia) -> DownloadedMedia:
        """Sync variant. Runs aprocess in a new event loop.

        Prefer aprocess in async contexts. This is provided for compatibility
        with the sync PostProcessor protocol used by DownloadRouter.
        """
        return asyncio.run(self.aprocess(media))

    async def _resolve_audio(self, media: DownloadedMedia) -> Path | None:
        """Return the audio file path, extracting from video if necessary.

        Resolution order:
        1. media.audio_file a-transcribe already an audio file, use directly
        2. Existing audio sidecar in MediaStorage a-transcribe skip re-extraction
        3. media.video_file a-transcribe extract audio, persist sidecar, return path
        4. None a-transcribe no audio source available
        """
        # Already have a dedicated audio file
        if media.audio_file is not None:
            return media.audio_file

        # Determine sidecar path
        sidecar_fp = self._storage.media_path(
            media.source, media.source_id, role="audio", ext=self._audio_format
        )

        # Reuse existing sidecar
        if sidecar_fp.exists():
            lg.debug(f"Reusing existing audio sidecar: {sidecar_fp}")
            return sidecar_fp

        # Extract from video
        if media.video_file is not None and needs_extraction(media.video_file):
            await extract_audio_async(
                media.video_file,
                sidecar_fp,
                audio_format=self._audio_format,
            )
            # Register sidecar in media's file list so DB layer persists it
            media.all_files.append(sidecar_fp)
            media.audio_file = sidecar_fp
            return sidecar_fp

        return None
```

## `build_router` update

`TranscriptionHook` now needs `storage` in addition to `transcriber`.
`build_router` already receives both:

```python
def build_router(
    config: DownloaderConfig,
    storage: MediaStorage,
    transcriber: BaseTranscriber | None = None,
) -> DownloadRouter:
    ...
    post_processors = []
    if transcriber is not None:
        audio_format = _audio_format_for(transcriber)
        post_processors.append(
            TranscriptionHook(
                transcriber=transcriber,
                storage=storage,
                audio_format=audio_format,
            )
        )
    ...


def _audio_format_for(transcriber: BaseTranscriber) -> str:
    """Pick the best audio extraction format for the given provider."""
    # OpenAI API: minimise upload size with mp3
    if transcriber.provider_name == "openai_api":
        return "mp3"
    # Local providers: wav is lossless, faster to write and load
    return "wav"
```

## Dependency

`pydub` is added to the `stt-*` optional groups, not to core deps. It is only
needed if transcription is enabled. `pydub` itself requires `ffmpeg` to be
installed on the system a-transcribe this is a runtime requirement, not a Python package,
and should be documented in the deployment notes.

```toml
stt-local      = [
    "llm-core[whisper] @ git+https://github.com/Pitrified/llm-core@v0.1.0",
    "pydub>=0.25",
]
stt-local-fast = [
    "llm-core[faster-whisper] @ git+https://github.com/Pitrified/llm-core@v0.1.0",
    "pydub>=0.25",
]
stt-api        = [
    "llm-core[openai] @ git+https://github.com/Pitrified/llm-core@v0.1.0",
    "pydub>=0.25",    # still needed for extraction before upload
]
```

Uses `FakeTranscriber` from `llm-core.transcription.testing.fake`:

```python
from llm_core.transcription.base import TranscriptionResult
from llm_core.transcription.testing.fake import FakeTranscriberConfig

def make_fake_transcriber():
    return FakeTranscriberConfig(
        responses=[TranscriptionResult(text="Boil the pasta.", language="it")]
    ).create_transcriber()
```

- `aprocess` with `media.audio_file` set a-transcribe no extraction, transcriber called once
- `aprocess` with `media.video_file` set, no sidecar a-transcribe extraction called, transcriber called
- `aprocess` with `media.video_file` set, sidecar already exists a-transcribe extraction skipped,
  transcriber called with sidecar path
- `aprocess` with no audio and no video a-transcribe returns unchanged, transcription is `None`
- `process` (sync) delegates correctly to `aprocess`

Mock `extract_audio_async` in the hook tests to isolate from ffmpeg.

## Open points

**ffmpeg on the box.** `pydub` requires `ffmpeg` as a system binary. It is almost
certainly already installed (yt-dlp also depends on it for format merging), but it
should be listed explicitly in the deployment prerequisites.

**webm from yt-dlp.** When yt-dlp downloads a YouTube video in webm format,
`AudioSegment.from_file(fp, format="webm")` works correctly a-transcribe pydub delegates
the decode to ffmpeg which handles webm natively. No special case needed.

**Large video files.** pydub loads the entire decoded audio into memory as a
`numpy`-backed buffer before exporting. A 30-minute video at 16kHz/16-bit mono
is roughly 60 MB of RAM a-transcribe acceptable on the linux box. For very long content
(full lectures, long-form YouTube) this could be a concern; note it as a future
optimisation candidate if needed.

**Sidecar registration in DB.** `media.all_files.append(sidecar_fp)` inside
`_resolve_audio` means the DB service will persist a `MediaFile` row for the
audio sidecar. The `role="audio"` distinguishes it from the video row in queries.
This is the intended behaviour.
